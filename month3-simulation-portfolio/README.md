# Month 3: シミュレーション & ポートフォリオ

## 概要

Physical AIエンジニアへの最終ステップとして、**NVIDIA Isaac Sim**によるシミュレーション環境の構築、
**強化学習（RL）の基礎**、そして3ヶ月間の成果を集約した**ポートフォリオの制作**に取り組む。

Month 1で構築したROS 2の基盤、Month 2で実装したAI統合パイプラインを、
シミュレーション環境上で動かし、最終的に「見せられる成果物」として仕上げるのがこの月の使命である。

---

## なぜMonth 3が重要なのか

### Sim-to-Real：現代ロボティクス開発の核心

現代のロボティクス開発において、**シミュレーション上で十分に検証してから実機に展開する**
（Sim-to-Real）アプローチは、もはやオプションではなく標準的な開発手法となっている。

| 観点 | シミュレーションの利点 |
|------|----------------------|
| **コスト** | 実機破損リスクゼロ、ハードウェア不要で試行錯誤可能 |
| **速度** | 数千回の試行を並列実行、現実の何倍もの速度で学習 |
| **安全性** | 危険なシナリオも安全にテスト可能 |
| **再現性** | 同一条件での繰り返しテストが容易 |
| **スケール** | ドメインランダマイゼーションで多様な環境を自動生成 |

NVIDIAはIsaac SimをPhysical AIの中核プラットフォームとして位置づけており、
Unitree、Boston Dynamics、Agility Roboticsなどの主要ロボット企業がこのエコシステムを採用している。

### ポートフォリオ：スキルの実践的な証明

Physical AI 領域でのスキルを示すには、
**動くデモ付きのポートフォリオ**が最も効果的である。

```
スキル証明の段階:

  「ROS 2を勉強しました」            → 知識レベル
  「Isaac Simでロボットを動かしました」 → 実践レベル
  「デモ動画付きのGitHubリポジトリです」 → 実装力の証明
```

Physical AI の現場では、
**実際に手を動かしてモノを作れるエンジニア**が求められている。
ポートフォリオはその最も直接的な証明である。

## ファームウェア経験が活きるポイント

ファームウェアの経験は、Month 3において特に強力なアドバンテージとなる。

### シミュレーション領域

| ファームウェアの知識 | Month 3での活用 |
|---|---|
| テスト治具・テスト環境の構築 | シミュレーション環境構築の設計思想が共通 |
| センサーの物理特性・ノイズ特性 | Isaac Simのセンサーシミュレーション設定に直結 |
| EMC試験・環境試験の経験 | ドメインランダマイゼーションの発想と同根 |
| ハードウェアのデバッグ手法 | Sim-to-Real gapの分析・解消に応用可能 |
| 回路シミュレーション（SPICE等） | 物理シミュレーションの概念が共通 |

### ポートフォリオ領域

| ファームウェアの知識 | Month 3での活用 |
|---|---|
| KiCad基板設計 | ボーナスプロジェクト：カスタムセンサーボード設計 |
| ドキュメント作成（仕様書、設計書） | 技術ブログ・README執筆のスキル |
| ハードウェア＋ソフトウェアの統合 | HW/SW両面を語れるユニークな強み |
| 製品開発のライフサイクル理解 | プロジェクト全体を俯瞰したポートフォリオ構成 |

**核心**: ハードウェアを深く理解しているエンジニアが、シミュレーションの「嘘」と「本物」を
見分けられるのは、ソフトウェアのみの経歴では得られない圧倒的な強みである。
「シミュレーションと実機の差をどう埋めるか」を語れることは、Physical AI エンジニアとして重要なスキルである。

## Month 3の目標

### 到達目標

このMonthを完了すると、以下のスキルを習得できる。

1. **NVIDIA Isaac Sim**をセットアップし、ロボットシミュレーションを実行できる
2. **センサーシミュレーション**（カメラ、LiDAR、IMU等）を構築できる
3. **ドメインランダマイゼーション**の概念を理解し、適用できる
4. **強化学習（RL）の基礎**を理解し、簡単なタスクを学習させられる
5. **Isaac Sim ↔ ROS 2ブリッジ**を構築し、データを連携できる
6. **ポートフォリオ用GitHubリポジトリ**を整備し、公開できる
7. **デモ動画**を制作し、成果をわかりやすく見せられる
8. **技術ブログ記事**を執筆し、アウトプットできる

**最終成果物**: ポートフォリオがGitHub上に公開され、デモ動画付きでスキルを証明できる状態

### Isaac Sim ↔ ROS 2 接続の仕組み

```
  Windows 11 (ホスト)                    WSL2 (Ubuntu 22.04)
  ┌────────────────────┐                ┌────────────────────┐
  │  NVIDIA Isaac Sim  │                │   ROS 2 Humble     │
  │  ┌──────────────┐  │   ROS 2       │  ┌──────────────┐  │
  │  │ Isaac Sim    │  │   Bridge      │  │ VLM Node     │  │
  │  │ ROS 2 Bridge │──┼──────────────>│  │ Nav2         │  │
  │  │ Extension    │  │  (DDS/UDP)    │  │ SLAM         │  │
  │  └──────────────┘  │                │  │ Control      │  │
  │                    │                │  └──────────────┘  │
  │  GPU: RTX 5070     │                │                    │
  │  (レンダリング+物理) │                │  GPU: RTX 5070     │
  │                    │                │  (AI推論)           │
  └────────────────────┘                └────────────────────┘
```

**注意**: Isaac SimとAI推論が同時にGPUを使用する場合、8GB VRAMの制約に注意が必要。
タスクに応じてGPU使用量を調整するか、時分割で使用する戦略を取る。
